로그인 페이지와 회원가입 페이지 모듈의 내부 로직 및 결함 분석

## 1. 테스트 대상
- 'lib/pages/login.dart' : Firebase Auth 기반 로그인 및 세션 상태 저장
- 'lib/pages/singup.dart' : 유저 정보 등록 및 Firestore 연동

## 2. 테스트 베이시스
- **화이트박스 테스팅** : 소스 코드의 조건문 및 데이터 흐름 분석
- **Firebase Auth 명세** : 이메일 형식, 비밀번호 최소 길이(6자) 등

## 3. 테스트 케이스 및 결함 분석
| ID | 테스트 시나리오 | 기대 결과 (Expected) | 실제 결과 (Actual) | 결과 | 유형 |
| **TC-01** | 이름 필드 비우고 회원가입 시도 | "이름을 입력해주세요" 스낵바 노출 |   | 기능 |
| **TC-02** | 이메일/비번 비우고 회원가입 시도 | 클라이언트 측 사전 차단 및 경고 |  
| **TC-03** | 중복 이메일로 회원가입 시도 | '이미 가입되어 있습니다' 다이얼로그|    |
| **TC-04** | 로그인 성공 시 세션 확인 | 로그인 성공 시 sharedPrefences에 상태 저장 |   
| **TC-05** | 비밀번호 5자 이하 입력 | '최소 6자 이상' 안내 메세지|
// Firebase 기본 설정은 보통 6자 이상

### [결함 1] 클라이언트 측 입력 유효성 검사 부재
- 현상 : 이메일과 비번이 빈 값일 때 클라이언트가 막지 않고 서버(Firebase)로 요청 보냄
- 영향 : 불필요한 네트워크 리소스 소모 및 사용자 피드백 지연
- 개선안 : 'signup()'함수 시작 부분에 'email.isEmpty'와 'pw.isEpmpty' 체크 함수 추가 

### [결함 2] 비밀번호 강도 체크 미비
- 현상 : 5자 이하 입력 시 서버 에러가 날 때까지 사용자 알 수 없음
- 영향 : 회원가입 시 반복적인 실패로 인한 사용자 이탈 가능성
- 개선안 : 비밀번호 입력창 하단에 "최소 6자 이상" 가이드 문구 노출시키거나 실시간 유효성 검사 기능 추가
