3/8 실습

**1. [인수 테스팅 | UI/UX 결함] 일정 제목 미입력 시 예외 처리 부재 {안내 부족}**
- 현상 : 일정 추가 다이얼로그에서 제목을 입력하지 않고 '저장'을 누르면 데이터는 저장되지 않지만 사용자에겐 아무런 피드백(경고 메세지 등)이 없음
- 영향 : 사용자는 앱이 멈췄거나 오류난 것으로 오해할 수 있음
- 개선안 : 제목을 입력하지 않을 경우 "제목을 입력해주세요"라는 스낵바나 메세지 노출 필요

**2. [인수 테스팅] 일정 추가 후 수정 불가 {수정 불가}**
- 현상 : 일정 추가 후 일정을 누르면 아무효과 없음, 수정할 수 있는 기능 존재하지 않음
- 영향 : 일정 시간 혹은 제목 수정하고 싶을 시 삭제 후 다시 일정 추가해야하는 번거로움 발생
- 개선안 : 일정 수정 기능 추가 필요

**3. [인수 테스팅] 일정 추가 버튼 위치 불편 {조작 간섭}**
- 현상 : 일정 추가 버튼이 네비게이션 바에 결쳐져 있음
- 영향 : '출석부' 버튼 누르기 불편 및 UI상 불편
- 개선안 : 일정 추가 버튼을 조금 더 오른쪽 하단 네비게이션 바 위 위치쪽으로 수정

**4. [시스템 테스팅] 음원 누를 시 시간 걸림 {성능 저하}**
- 현상 : 음원 누를 시 음원이 나올 때까지 4초 정도의 시간 걸림
- 영향 : 음원이 누른 즉시 나오지 않아서 눌렀는지 안 눌렀는지 알 수가 없음
- 개선안 : 로딩 ui 추가 및 스트리밍 최적화

**5. [인수 테스트] 음원 화면 구성 불편 {레이아웃 침범}**
- 현상 : 음원 화면이 네비게이션 바 구역까지 나타남
- 영향 : 네비게이션 바에 걸쳐 있는 음원을 틀기 어려움
- 개선안 : 네비게이션 바 위로 화면이 나타날 수 있도록 함

**6. [인수 테스트] 음원페이지 배속 버튼 구성 {ui 정렬 불량}**
- 현상 : 배속 버튼 크기가 커서 두줄로 버튼이 밀려 나오게 됨
- 영향 : 배속 버튼 크기 때문에 ui 불편
- 개선안 : 배속 버튼 크기 조절로 ui 정리

**7. [인수 테스트] 공휴일 및 토요일, 일요일 날짜 색 구분량 {시각적 구분 부재}**
- 현상 : 현재 주중과 주말 및 공휴일 텍스트 색깔이 동일
- 영향 : 언제 쉬는지, 주일이 언제인지 한눈에 확인 어렵
- 개선안 : 공휴일 및 주말 색깔 버전 소스를 가져다 사용
